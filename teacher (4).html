Add a student

Each student gets their own HTML file in the repo

How to add a new student

Each student has their own studentname.html file in the GitHub repo. Copy sandugash.html, rename it, and update the student name and Supabase config at the top of the file.

1
Duplicate sandugash.html in your repo and rename it to the student's name, e.g. aisha.html
2
Open the file and change STUDENT_NAME at the top of the script to the new student's name
3
Push to GitHub — the student gets their own link: yoursite.github.io/aisha.html
4
Add the student's name to the sidebar of this teacher dashboard

Connection setup

Connect to Supabase so homework results sync automatically

Without this setup, homework results only live in the student's browser and you can't see them. This takes about 5 minutes and is completely free.

Step 1 — Create a free Supabase project

Supabase is a free database that both the student site and this teacher dashboard connect to. No credit card needed.

1
Go to supabase.com → click "Start your project" → sign in with GitHub
2
Create a new project, name it anything (e.g. "english-class")
3
Wait ~1 min for it to set up, then go to Settings → API
4
Copy your Project URL and anon public key

Step 2 — Create the results table

In your Supabase project go to SQL Editor and run this query once:

CREATE TABLE hw_results ( id SERIAL PRIMARY KEY, student TEXT NOT NULL, hw_number INTEGER NOT NULL, score_pct INTEGER NOT NULL, correct INTEGER NOT NULL, total INTEGER NOT NULL, vocab_mastered INTEGER DEFAULT 0, vocab_practicing INTEGER DEFAULT 0, vocab_review INTEGER DEFAULT 0, submitted_at TIMESTAMPTZ DEFAULT NOW() ); -- Allow public inserts and reads (student site is public) ALTER TABLE hw_results ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON hw_results FOR ALL USING (true) WITH CHECK (true);

Step 3 — Enter your Supabase credentials

Paste your credentials below. They are saved in your browser and used to fetch results on this dashboard.

Step 4 — Add credentials to the student file

Open sandugash.html in your repo, find the CONFIG section near the bottom of the <script> tag, and paste your Supabase URL and key there. This allows the student's submissions to be saved.

// ── SUPABASE CONFIG (add your credentials here) ── const SUPABASE_URL = 'https://xxxx.supabase.co'; const SUPABASE_KEY = 'eyJh...your-anon-key...'; const STUDENT_NAME = 'Sandugash';

Once you add these and push to GitHub, every HW submission will automatically save to Supabase and appear on this dashboard.
